# テストコードにおける記述規則

記述規則は原則、[Microsoftの公式ドキュメント](https://docs.microsoft.com/ja-jp/dotnet/core/testing/unit-testing-best-practices)に従うものとする。  

ここでは、更に厳密な規則を記述する。


# アセンブリ

## 粒度

アセンブリの分割粒度は、テスト対象と同じにする。

## 命名

アセンブリの命名は、以下の規則に従う。

```text
(テスト対象のアセンブリ名).Test
```

---

# ディレクトリ

## 構造

ディレクトリ構造は、テスト対象と同じにする。

## 命名

ディレクトリの命名は、テスト対象と同じにする。

---

# 名前空間 `namespace`

## 構造

名前空間の構造は、テスト対象と同じにする。

---

# クラス `class`

## 粒度

クラスの粒度は、テスト対象のクラス1つにつき、1つとする。

## 命名

クラスの命名は、以下の規則に従う。

```text
(テスト対象のクラス名)Test
```

---

# リージョン `region`

## 粒度

リージョン `#region` の粒度は、テスト対象のメソッド1つにつき、1つとする。

## 命名

リージョン `#region` の命名は、テスト対象のメソッドと同じにする。

---

# メソッド

## 粒度

メソッドの粒度は、テストケース1つにつき、1つとする。

## 命名

メソッドの命名は、以下のフォーマットに従う。

```text
(テスト対象のメソッド名)_(テストケースの説明)_(期待する結果)
```

以下によく使われる表現を示すが、必ずしもこれに従う必要はない。  

過度に詳細な命名は、しばしばテストの実装内容をそのまま表現してしまう。その上、冗長な構文の読解を放棄するリスクを考えれば、短い文章で理解を求めた方が良い。  
そのため、最低限意図が伝わればよいとする。


### テストケースの説明

| 表現               | 命名                       |
|------------------|--------------------------|
| 常時               | `Always`                 |
| `Hoge`のとき        | `WhenHoge`               |
| `Hoge`が`2`のとき    | `WhenHogeIs2`            |
| `Hoge`が`2`でないとき  | `WhenHogeIsNot2`         |
| `Hoge`が`2`以上のとき  | `WhenHogeIs2OrGreater`   |
| `Hoge`が`2`を超えるとき | `WhenHogeIsGreaterThan2` |
| `Hoge`が`2`以下のとき  | `WhenHogeIs2OrLess`      |
| `Hoge`が`2`未満のとき  | `WhenHogeIsLessThan2`    |


### 期待する結果

| 表現                 | 命名                   |
|--------------------|----------------------|
| `A`を返す             | `ReturnsA`           |
| 変数`A`が`2`となったものを返す | `ReturnsThatVarAIs2` |
| `A`を`throw`する      | `ThrowsA`            |


### 汎用的な表現

| 表現               | 命名                       |
|------------------|--------------------------|
| 定数`A`            | `ConstA`                 |
| 変数`A`            | `VarA`                   |
| 引数`A`            | `ArgA`                   |
| `Hoge`かつ`Fuga`   | `HogeAndFuga`            |
| `Hoge`もしくは`Fuga` | `HogeOrFuga`             |

---

# 実装

実装は、以下の3つの構造に分ける。

## テストケースを再現するための準備

テストケースを再現するために、オブジェクトの初期化を行う。  

静的なヘルパーメソッドを呼び出しても良い。

## テスト対象のメソッドの実行

テスト対象のメソッドを実行し、変数 `actual` にその結果を格納する。  

ただし、例外の `throw` を期待するテストケースの場合、この工程をスキップして直接 `Assert.That()` を呼び出してよい。

## 実行結果の検証

`Assert.That()` を使用して、実行結果を検証する。
